Hard Problems Solved

What Makes This Technically Impressive

Each of these problems typically requires dedicated SaaS tools, custom middleware, or expensive developer time to solve. We solved all of them in a single, unified codebase.

Agency: $3-5K

⚡ Real-Time Payment → Trainerize Sync

Square processes recurring payments automatically, but Trainerize has zero native integration with Square. The trainer had no way to know which clients have paid, failed, or lapsed.

Solution: Cloudflare Worker catches 6 Square webhook events (subscription.updated, payment.completed, payment.failed, invoice.*), cross-references the customer via Square Customers API, then writes tags + notes + nudge notifications to Trainerize — all in under 200ms.
Agency: $2-3K

💰 Zero-Config Plan Discovery

Hardcoding training plans means every price change, new plan, or removed plan requires a code deployment. For a trainer managing their own site, this is a non-starter.

Solution: Auto-discovers ALL plans from Square Catalog API. Parses plan names, durations, and pricing variations using intelligent pattern matching. Add a plan in Square Dashboard → website updates automatically. 24-hour cache with graceful fallback to local data.
Agency: $1.5-2.5K

🔔 Non-Persistent Push Notifications

Trainerize doesn't support transient payment notifications. Sending a message creates a permanent chat entry, cluttering the client's conversation history with billing noise.

Solution: Invented a "nudge tag" pattern — Worker adds a temporary tag (e.g. nudge:payment-due), which fires a Trainerize Automation push notification, then the tag is cleaned up on the next webhook. Client sees a phone notification, not a persistent message.
Agency: $2-3K

👥 Automatic Client Provisioning

When a client purchases a plan, the trainer would normally have to manually create them in Trainerize, assign a program, set credits, and email login details. This takes 10-15 minutes per client.

Solution: One payment triggers a 4-step provisioning pipeline: create client (auto-sends invite email) → assign training program → log payment record → set session credits with expiry. Zero manual work.
Agency: $1-1.5K

🏆 Challenges Auto-Sync & Expire

Fitness challenges need to appear on the website, disappear when ended, and be manageable from multiple sources (admin panel, Zapier, Trainerize webhooks).

Solution: Cloudflare KV stores challenges with full CRUD API. Frontend auto-filters by date — ended challenges vanish automatically. Admin interface at ?admin=challenges, plus webhook endpoint for Zapier/Trainerize automation.
Agency: $0.5-1K

🔌 Sandbox/Production Auto-Switch

Square has completely separate sandbox and production environments with different API URLs, SDKs, and credentials. Most implementations hardcode one or the other.

Solution: Single line of logic detects the environment: IS_SANDBOX = appId.startsWith('sandbox-'). API base URLs, Web Payments SDK, and all config auto-switches. Change one env variable to toggle environments.
